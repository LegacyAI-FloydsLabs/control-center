# ATerm

The self-aware terminal. Structured output intelligence for AI agents. Power UX for humans.

## What ATerm Does

ATerm is a terminal emulator that **understands its own output**. Instead of raw byte streams, agents get:

- **Semantic state:** `ready` / `busy` / `error` / `waiting_for_input` / `exited` — with confidence scores
- **Distilled output:** 5 modes from raw (0% reduction) to structured (70% reduction)
- **Output marks:** Numbered anchors — agents say "fix mark 3" instead of describing errors in prose
- **Progressive disclosure:** 3B models get simple responses. Frontier models get full traces. Same API.

ATerm runs in the browser and exposes an MCP server. Any agent harness can connect.

## Quick Start

```bash
# Clone and install
git clone https://github.com/LegacyAI-FloydsLabs/aterm.git
cd aterm
bun install
cd ui && bun install && cd ..

# Fix node-pty permissions (macOS)
chmod +x node_modules/node-pty/prebuilds/darwin-arm64/spawn-helper

# Start the server
npx tsx src/server.ts
```

ATerm prints a URL with your auth token:
```
ATerm v0.1.0
Port:  9600
Token: <auto-generated>
URL:   http://localhost:9600?token=<token>
```

Open the URL in your browser. Create a session. Type commands.

## MCP Server Setup

### Claude Code

Add to your MCP config (`~/.claude/mcp.json` or project `.claude/mcp.json`):

```json
{
  "mcpServers": {
    "aterm": {
      "command": "npx",
      "args": ["tsx", "/path/to/aterm/src/mcp/server.ts"],
      "env": {
        "ATERM_TOKEN": "<your-token-from-.aterm-token>"
      }
    }
  }
}
```

### Cursor / Any MCP Client

Same pattern — point at `src/mcp/server.ts` with the `ATERM_TOKEN` env var.

### Available MCP Tools

| Tool | Description |
|------|-------------|
| `aterm_list` | List all sessions with semantic state |
| `aterm_create` | Create a new terminal session |
| `aterm_run` | Send command, wait for result, return distilled output |
| `aterm_read` | Read session output with distillation mode |
| `aterm_start` | Start a stopped session |
| `aterm_stop` | Stop a running session |
| `aterm_cancel` | Send Ctrl+C |
| `aterm_answer` | Reply to a prompt (y/n, password, etc.) |
| `aterm_delete` | Delete a session |
| `aterm_note` | Read/write session scratchpad |
| `aterm_search` | Search scrollback across all sessions |
| `aterm_history` | Get command history |
| `aterm_broadcast` | Send input to multiple sessions |

## HTTP API

Single endpoint: `POST /api/do`

### Tier 1 (any model)
```json
{"action": "run", "session": "build", "input": "make all"}
```

### Tier 2 (adds wait patterns)
```json
{"action": "run", "session": "build", "input": "make all", "wait_until": "Build complete", "timeout": 120}
```

### Tier 3 (full intelligence)
```json
{"action": "run", "session": "build", "input": "make all", "output_mode": "structured", "include_marks": true, "include_advanced": true}
```

All 17 actions: `list`, `read`, `run`, `stop`, `start`, `cancel`, `answer`, `create`, `delete`, `note`, `search`, `broadcast`, `history`, `checkpoint`, `record`, `verify`, `batch`.

```yaml
  - name: tests

```

```bash
npx tsx src/server.ts --config=aterm.yml
```

## Security Notice

ATerm is designed for **local development use**. The auth token provides access to terminal sessions on the host machine.

- Token is auto-generated (256-bit) and written to `.aterm-token` with 0600 permissions
- Pass token via `Authorization: Bearer <token>` header or `?token=<token>` query param
- **Do not expose ATerm to untrusted networks** without additional authentication (reverse proxy, VPN, etc.)
- The token grants full terminal access to the host — treat it like an SSH key

## Checkpoint Honesty

The `checkpoint` action saves: scrollback content, environment snapshot, working directory, command history, scratchpad notes, and session config.

It does **NOT** save:
- Running process state (PIDs, open file descriptors, network connections)
- Build caches, Docker containers, or database state
- Anything outside the terminal session itself

A restore kills the current PTY, restarts with saved config, and replays the saved scrollback visually. It's a workspace snapshot, not a VM snapshot.
