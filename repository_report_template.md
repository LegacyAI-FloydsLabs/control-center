# # Repository Report Template

Copy and customize this template for your project:

## JSON Format (Recommended)

```json
{
    "project_name": "Your Project Name",
    "completion_percentage": 45,
    "tech_stack": ["React", "Node.js", "MongoDB", "AWS", "TypeScript"],
    "complexity_score": 7,
    "team_size_minimum": 3,
    "go_to_market_timeline": "3 months",
    "industry_vertical": "Technology",
    "business_model": "B2B SaaS",
    "technical_debt": 30,
    "scalability_needs": "medium",
    "target_users": "Small to Medium Businesses",
    "key_features": ["Feature 1", "Feature 2", "Feature 3"],
    "risks": ["Risk 1", "Risk 2"]
}
```

## Markdown Format (Alternative)

```markdown
# Project: Your Project Name

- **Completion**: 45%
- **Tech Stack**: React, Node.js, MongoDB, AWS, TypeScript
- **Complexity**: 7/10
- **Timeline**: 3 months to launch
- **Industry**: Technology
- **Business Model**: B2B SaaS
- **Target Users**: Small to Medium Businesses
- **Key Features**: Feature 1, Feature 2, Feature 3
```

## Field Explanations

- **project_name**: Name of your SaaS product
- **completion_percentage**: 0-100, how much is built
- **tech_stack**: List of technologies used
- **complexity_score**: 1-10, technical difficulty
- **go_to_market_timeline**: Time until launch
- **business_model**: B2B, B2C, or B2B2C
- **scalability_needs**: low, medium, or high
- **technical_debt**: 0-100, amount of refactoring needed

-------
## Mandatory execution contract

For EACH requested item:
1) Show exact action taken
2) Show direct evidence (file/line/command/output)
3) Show verification result
4) Mark status only after proof

## Forbidden behaviors

- Declaring "done" without evidence
- Collapsing multiple requested items into one vague summary
- Skipping failed steps without explicit blocker report

## Required output structure

A) Requested items checklist
B) Per-item evidence ledger
C) Verification receipts
D) Completeness matrix (item -> done/blocked -> evidence)

## Hard gate

If any requested item has no evidence row, final status MUST be INCOMPLETE.
